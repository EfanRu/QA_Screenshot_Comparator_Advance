## Записи по диплому

####    Предусловия:
- Data set:
  - будем брать 50 страниц и 10 раз менять размер скриншота (500 кейсов), моделируя разные размеры экранов. Основная боль при 
  сравнении скриншотов в том, что у всех разные экраны и скриншоты соответственно разные. Но это легко обходится 
  при тестировании в docker container. Но мы должны это учитывать при обучении модели. При этом мы можем сравнивать 
  их друг с другом, тогда будет 10! / (2! × 8!) = 45 уникальных пар на 1 страницу. Всего: 50 * 45 = **2250 уникальных пар**;
  - В каждой из пар можно будет внести изменения в html код для имитации багов и допустимых искажений **2250 * 2 = 4500 пар**:
    - Баг: Изменение значения текста в кнопках | заголовках | тексте и сдвиг формы  или объектов на значительное 
    расстояние. Чтобы добавить уникальности, данные для изменений будут генерироваться случайным образом;
    - Допустимые искажения: сдвиг формы на несколько пикселей, актуально при едином фреймворке фронтенда и случается
    при незначительных изменениях - **Основная проблематика**;
    **Ограничения для MVP - буду менять только то, что есть на каждой странице.** Слишком трудозатратно делать это в ручную.
  - Собирать данные буду автоматически, так как их нужно много для хорошего обучения и диплома, с учетом моего уровня ML.
  Плюсы: сделав 1 раз автоматизированный шаблон, то можно будет его переиспользовать для других проектов. 
  Минусы: плохое разнообразие негативных кейсов. Шанс, что модель запомнит 5 приемов, хотя в них будет добавлены случайности.
  - **Итог по Data set**: соблюдается баланс ( >> 1) между = (количеством данных для обучения) * 0,(искустенную генерацию данных)
  / (трудозатратами на автоматизацию - польза от автоматизации).
